# LifeQuest 测试与代码审计计划

> 执行顺序：先做代码审计，再跑自动化回归，最后用 Playwright 验证真实浏览器主流程。发现问题先记录，不要跳过。

**目标：** 验证当前仓库的后端 API、前端交互、页面路由和数据一致性；同时审计高风险代码路径，找出潜在 bug、回归和安全问题。

**范围：**
- 后端：`backend/app/api/`、`backend/app/services/`、`backend/app/repositories/`、`backend/app/schemas/`
- 前端：`frontend/src/router/`、`frontend/src/stores/`、`frontend/src/services/`、`frontend/src/views/`、`frontend/src/components/`
- 测试：`backend/tests/`

## 1. 代码审计

### 后端审计点
- `auth.py`：登录、注册、JWT 过期、401/400 返回是否一致。
- `todo.py`、`notes.py`、`shop.py`、`backpack.py`、`achievement.py`：服务层是否正确提交事务、刷新对象、更新金币/经验/库存/历史记录。
- `repositories/`：是否存在遗漏权限校验、重复查询、静默失败或不一致提交。
- `config.py`：环境变量、`SECRET_KEY`、数据库路径、默认值是否安全。
- `schemas/`：输入输出校验是否覆盖空值、超长值和非法格式。

### 前端审计点
- `router/index.js`：未登录跳转、已登录访问登录页的重定向、深链路访问是否正常。
- `services/api.js`：Token 注入、401 自动退出、异常是否会误跳。
- `stores/auth.js`：登录/注册/退出后的状态同步是否一致。
- 各 `views/`：表单校验、加载态、按钮禁用、空状态、成功/失败提示、中文文案是否正常显示。
